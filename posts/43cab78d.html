    MacOS拷贝iphone中的照片

    标签： Macos 分类： 操作系统 创建时间：2020-11-22 01:28:56 更新时间：2025-04-28 14:37:40

    前言

    试了很多方法，最后还是选择了安装windows10 虚拟机，然后在虚拟机中安装iTools软件，将iphone通过usb连接到电脑，然后挂载到虚拟机中，使用iTools工具进行图片的导出，然后在使用VM Fusion自带的共享文件夹的方式进行文件的拷贝，比使用MacOS自带的照片工具都好使，可以将文件想移到哪里就移到哪里，也不用在照片中进行各种操作才能移动文件了。

    1.使用访达同步数据

    这就是将iphone用usb和mac链接之后，然后打开访达看到的内容，这和我想要的不是一回事啊。我就想将图片，视频等内容从手机上倒到电脑上，这里又是备份，又是干啥的，不知道所以然。

    经过我多次的尝试之后，其实这个访达是用来进行整机备份的，包括图片啊，视频、App等所有的内容，可以进行备份也可以进行恢复备份。备份的数据当然不能直接查看啦，毕竟是加密之后进行的备份。

    我不知道是因为我的数据线有问题还是怎么得，经常性的出现要解锁iphone的提示，但是我明明已经将iphone解锁了。

    2.使用Mac照片App

    后来找到了照片App这个工具进行导入图片。

    (1) 用 USB 连接线将 iPhone、iPad 或 iPod touch 连接到 Mac。
    (2) 打开“照片”App。
    (3) “照片”App 会显示“导入”屏幕，其中包含所连接设备上的全部照片和视频。如果没有自动显示“导入”屏幕，请点按“照片”边栏中对应设备的名称。
    (4) 出现提示时，使用密码来解锁 iOS 设备。如果 iOS 设备上出现提示，要求您信任这台电脑，请轻点“信任”以继续。
    (5) 您可以选择要导入的照片，然后点按“导入所选项”，或点按“导入所有新照片”

    (6) 等待这个过程完成，然后断开设备与 Mac 的连接。
    我在导入之后，中间有一次断掉了，不知道什么原因导致的，剩下的总是导入不进入了。

    经过实际测试，每次重开机Mac之后，就能导入一批新的数据，导入完了一批之后，剩下的部分内容，只能等下一次开机再次导入才可以了。

    5.使用Windows10

    在windows上安装iTunes,windows自带的照片工具，可以实现照片的复制。

    (1) 安装iTunes，直接在应用商店中搜索并安装iTunes就可以了。到底要不要装，其实我也是不知道了，如果不需要装，可以直接使用windows自带的照片工具导入试试。

    (2) 打开iTunes，然后找到一个手机的图标，可以点击进入，查看。

    (3) 打开windows自带的照片工具，选择右上角的导入->从USB导入，我尝试了很多次，,总是显示无法链接，最后才终于查到了我的iphone手机

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
