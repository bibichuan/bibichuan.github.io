    标签： 无 分类： 软件 创建时间：2019-08-05 07:23:49 更新时间：2025-04-28 14:37:41

    1.FastCopy

    如果在windows系统中进行复制，使用右键单击然后复制粘贴，虽然简单方便，但是有时候也会让人崩溃，比如有些项目中包含node_modules文件夹，但是这个文件夹里面的文件特别的小，windows在拷贝小文件时是尤其耗时间的，如果复制的时候，选择性的复制某些文件怎么办呢？这个时候就需要借助其他的软件进行快速的复制，比如FastCopy软件。该软件不仅可以快速复制文件，而且可以排除特定的文件或者文件夹，并且可以更加细粒度的控制粘贴覆盖的方式，比如说，判断两者大小不一样，就进行覆盖，或者判断两个文件修改日期不一样，但是大小相同也进行覆盖等，简直神器有没有？

    (1) 排除固定文件夹
    这里就以排除 node_modules 文件夹为例，勾选下方的Filter选项，就可以在Include和Exclude输入框中输入内容了，在Exclude输入框中输入“node_modules;”，选择源文件夹，和目标文件夹，点击Execute执行就可以了。复制过程中，自动掠过了node_modules文件夹，多个条件用分号分隔。

    参考文章：
    1.FastCopy包含和排除文件夹处理

    2.WinRAR

    (1) 排除压缩文件
    打开winrar压缩，选择文件,设置排除的文件，通常我们不想压缩node_modules文件夹下的内容，那样太大了。在要排除的栏目中，输入目录：\node_modules,多个目录使用空格分隔。如果路径包含空格，使用引号括住它。在不压缩直接存储的文件中，输入:``.rar *.zip *.jpg``

    (2) 选择用户User->Keys，可以添加相应的公钥

    把本机下的C:\Users[用户名].ssh\id_rsa.pub中的内容，复制到D:\ICW\home[用户名].ssh\authorized_keys文件中，没有authorized_keys就新建一个。

    4.iperf

    Iperf是一个网络性能测试工具，目前支持linux、windows、macOS操作系统，可以用于测试网络tcp、udp传输的最大带宽、网络的延迟及丢包率，我们利用iperf可以得到相对客观的网络质量以及网络设备性能的测试结果。在windows上使用，下载的文件都是一样的，包括了iperf3.exe 和cygwin1.dll，启动方式主要分为客户端和服务端，在服务端启动监听，然后在客户端使用命令进行测试。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
