    MacBookPro使用体验

    标签： Macos 分类： 操作系统 创建时间：2020-10-25 02:04:20 更新时间：2025-04-28 14:37:40

    总得来说体验就是：
    MacBook Pro，可以买，但是没有必要！
    AirPods Pro可以买，但是没有必要！
    功能很强大，但是不一定见的你能用到

    2020年9月27日

    2020年9月27号，趁着教育优惠的最后几天，我还是花了20852.00大洋，买了MacBook Pro 2020款(虽然是2020年买的，但是因为没有发布新品，估计也就算是2019款吧)，到今天10月25写文章的时候，已经过去了整整一个月的时间。我买的配置是i7+32G内存+512G硬盘+AMD Radeon Pro 5300M 4G显存(这个我没有找到在哪里现实是否有4G显存)。

    刚开始(9月21号)我其实选配的是i9版的，i9+16G内存+1T硬盘+AMD Radeon Pro 5500M 4G显存。第一次付款的时候，我其实还因为没有及时付款，导致了超出数量限制的问题，还是打电话给客服，让客服帮忙处理了无效的订单，我才能使用第一个学生身份购买了第一台电脑。我纠结了很久，查了很多的资料，其实我觉得CPU没有必要搞这么高的，而且硬盘也不需要这么大的，于是我在收到货的第二天，就申请了退货，然后又找了另外的在校学生的信息，换成了现在的配置。退货倒是挺快的，顺丰上门取货，也没有要运费，收到货之后，退款也很快到账了。

    第一次我用的是花呗免息分3期，通过额度提升的方式，最终凑到了21000元钱，但是由于退货了，花呗分期不能用了，也没有那么多的额度了。我自己的Apple ID也不嫩用了。第二次购买的时候，幸好我还有一张宁波银行的五万的信用卡，于是先用这个钱垫上了。至于Apple ID也是使用的别人的账号，收获手机号也是别人的，学生信息也是别人的，邮箱也是别人的，唯一相同的就是收货地址还是第一次的收获地址。不知道为什么，从我27号付款买下了第二台电脑之后，预计发货时间显示的时候10月15号，难道是因为期间有一个十月一假期吗？

    十月一的八天假期很快就过去了，也上了很多天的班，突然有一天，朋友告诉我，货到了，我还以为这么快么？因为我当时还在纠结要不要退掉，毕竟还是很贵的啊，虽然我现在月入18k,但是还是觉得好困难啊，买一台电脑。等我回到家，拿了快递，发现竟然是赠送的耳机AirPods Pro先到了。我使用教育优惠购买了电脑，而且送了一个AirPods耳机，但是我还是多花了900块钱，升级到了AirPods Pro，后来其实感觉没有必要。
    又过了几天，一直到10月17号，朋友终于给我消息了，说是电脑发货了，从发货到收货，当天就到了。下午下班，我连班都不加了，就跑回了宿舍去拿快递。拿到快递，我已经没有很愉快的心情了，总是在纠结要不要，要不要要不要要不要退回去，毕竟都是钱啊。犹豫了好多天，拿到东西后，也没有立刻就拆开，还是犹豫着，这就是没钱的毛病，犹豫ing….内存是不是太大了，32G要贵两千多块钱啊？耳机其实没有必要加钱买Pro的，用赠送的不是就好了？买不买AC+呢？万一坏了呢？

    后来，在17号晚上的七点半，我沐浴更衣之后，还是打开了电脑，主要是因为要退，还要退耳机，退了就没有教育优惠了，要想买，又得等一年，而且我的联想的电脑真的是撑不住了，32G内存，我可以开虚拟机学习Hadoop啊。我还特地的用买的相机拍了一个开箱视频，希望有一天能剪辑一下，放到B站上。

    刚开始，也没干啥，就是先把需要的软件安装一下，比如微信、QQ、钉钉、还有开发必备的VScode,Idea,Git,NVM,Nodejs,Hexo等等，当然安装也不是一帆风顺的，虽然命令行使用起来和Linux差不多，但是还是有些不同的，至于习不习惯呢，作为一个爱折腾的程序猿，也还说得过去的，主要就是因为墙的存在，让我着实废了一番功夫安装Git,以及NVM等。后来安装Adobe CC 2019也是废了一番功夫，主要就是破解麻烦。其他的都还好说的，吐槽一下，苹果的字体看起来都好小啊。另外就是软件全屏的时候，地下的程序坞就被盖住了，这个我不喜欢，像window上就还比较符合习惯，或者是Deepin也行的。
    除了系统使用之外，其实整个16寸的笔记本，外观还是挺漂亮的，缺点就是没有USB接口，现在还不能外接鼠标，需要买扩展坞什么的。屏幕比较大，色彩吗，好像也还说的过去。音响的质量也不错，有种立体声的感觉。
    至于耳机吗，到手后，就转手送人了，自己也没有怎么尝试过，体验过，本来是想卖掉的，还能回个血。其实我觉得买个华为的耳机也不错的，毕竟我花了100块钱买的蓝牙耳机，除了造型不好看，在耳朵里总是掉，其他的音质什么的，都没有啥毛病，对于我只是听听喜马拉雅来说，这完全够了，根本没有必要花这么多钱每买一个耳机，就算买耳机，买个低配的送的AirPods就好了，也没有必要非得加钱换个高端的，毕竟自己穷啊。

    所以总结起来，我使用Mac的感受，就是可以买，但是没有必要，性价比还是比较低的。说是做设计用Mac比较好，但是实际上，Windows又不是不能做设计。做一个前段，写代码，其实大部分的时间都是和浏览器打交到，所以也没有必要非得上一个这么庞大的系统。对内存来说，还是要上大点的内存，这我才开了几个软件啊，就已经快到了13G了，想想公司给我配的电脑是16G的都不够用的，也该是时候配个大点的内存了。

    2020年11月05日

    可能因为穷吧，我觉得确实对于提高生产力来说，MacBook Pro并没有想象中的那么大，很多人鼓吹说MacBook Pro能带来巨大的生产力的提升，我觉得有些言过其实了。而且最重要的一点，其实就是MacBook Pro提供的接口太少了，只是提供了雷雳3的四个接口，可是现在很多的外设还是USB接口的啊，比如移动硬盘和鼠标，这就让人非常的尴尬了，还要花钱买扩展坞，当然了，既然已经花了两万买了笔记本，至于说是小小的扩展坞，就不要心疼了吧。

    2020年11月08日

    今天我第二次听到了风扇的声音，第一次是我使用Nodejs编译程序的时候，今天这次，是我在虚拟机中运行一个3d游戏的时候，风扇声音特别的大，比我的老古董联想都要大。平常的时候基本上是没有风扇的声音的，因为大部分的时间都是在写文字，写博客，很少有运行大型程序的时候。今天我看了，MacBook pro 用的是i7-9750H,根本不是i7-9900k，怎么感觉被骗了呢。简直了，想要追求极致的体验，却搞得这么的狼狈。

    2021年01月30日

    苹果新一代的M1笔记本已经出来一个多月了，各种好评的声音是充斥着整个互联网，除了在软件适配上面有些问题，性能不输任何一代苹果以前的MacBook,也就是说，比16寸的也丝毫不差，这让我真是有些不舒服啊，哪怕我再等等，再等一个月，就可以用到了价格低一半，但是性能却丝毫不差的笔记本啊，新的笔记本，只需要10000块啊，只需要10000块，简直让人抓狂。现在卖掉，可能也来不及了，也不会有人买这么贵的电脑了吧，这个新的M1出来，可能我这个掉价，掉的就非常的厉害了。还是那句话，得不到的，永远在骚动。

    2022年01月04日

    买了一个绿联点扩展坞，鼠标买了一个爱国者的，差到其他地方这个无线鼠标都是好好的，但是就是插在了扩展坞上，就是总是用一段时间，就无响应了。后来我看了看晚上的答案，说是只要插着充电口，就会对集线器有影响，因为我总是喜欢插着电源工作，于是我尝试把电源拔掉，后来然后重新插入扩展坞，好像还真是解决了这个问题。

    参考文章:
    1.MacBook pro 13‘扩展坞出现的各种问题 终于看到一个非常考虑的回答，就是MacBook这个充电插口对集线器有影响。比如我在集线器上插上充电电源的时候，重新插拔集线器，集线器上的usb接口就都不能用了，但是如果没有在集线器上插电源的话，重新插拔集线器对usb接口的相关外设就没有影响
    2.2020 MacBook Pro 扩展坞接口失灵解决办法 同事跟我说是因为最新版本系统的问题，让我把电源拔掉，只插扩展坞试试， 我试了一下，立刻就连接上了， 很稳定。应该是这次Mac系统升级出现的问题，导致电脑上只能插一个接口，其他的接口会出现失灵的情况。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
