    在安装和配置Windows的时候，总会遇见各种各样的问题和毛病，随手一记，减少出错的概率。

    1.UEFI用光盘安装win10正版，出现缺少计算机所需的介质驱动程序。

    戴尔5820电脑解决方法是：在BOIS设置中，System Configuration->SATA Operation将SATA模式RAID ON改为AHCIi模式。

    3.修复系统引导

    在我重装之前呢，用了easyBCD准备修复windows和Deepin系统的双系统的引导，结果虽然安装成功了，但是每次开机还是用的Deepin的gurb引导。最后使用老毛桃软件PE，进入了PE系统，使用diskgenius软件，先是把各个分区的mbr重建了一遍，不管事。解决方式是要把windows所在的分区设置成活动分区，然后通过PE中的bootice修复引导，成功的。

    通过diskgenius设置windows所在分区为活动分区，然后使用boottice修复引导。

    4.开机显示“invalid partition table”

    在我重装win10的过程中，在系统重启的过程中，出现了“invalid partition table”，以前安装win10的时候是不会出现的，但是我用F12打开系统启动项的时候，Windows Boot在UEFI选项里面，所以我把BOIS里面的启动模式由Legacy模式改为了UEFI模式。很奇怪，上次安装系统的时候，并不会出现这个错误。

    9.无法安装在具有MBR分区的硬盘上

    我帮别人安装系统的时候，原有的系统是win7,我的U盘安装盘是win10，所以在安装的时候就会提示，无法安装在所选的盘中，必须安装在GPT盘中。但是从MBR转为GPT，是要格式化整个硬盘的，这是我所不希望看到了。

    解决方法，就是把安装盘里的bootmgr.efi文件以及efi文件夹删掉，然后再启动U盘安装系统，如果报错找不到efi，就直接回车继续就好了，还是可以继续再原有的C盘中安装系统的。

    2.Windows无法安装到磁盘磁盘具有MBR分区表的解决 (这个是转换硬盘的分区格式)

    10.这可能是由于CredSSP加密数据库

    新装了一台winserver2016的服务器，使用的是开启了远程桌面，但是从其他电脑连的时候，出现了“这可能是由于CredSSP加密数据库”这个错误，不是通常意义上的Orcal加密修正，问题

    12.Win10提示无法创建新的分区也找不到现有的分区解法

    我在含有两个硬盘的电脑上安装系统的时候，谈出了这个问题。第一块硬盘安装了Deepin系统，第二块硬盘是机械硬盘，本来有一个win10系统，想着重装的，可是使用u盘启动盘安装的时候，出现了这个问题。

    【解决】
    在A硬盘上安装Deepin系统，在B硬盘上安装Window系统，因为我不能重新格式化B这个硬盘，也不能修改分区mbr为gpt，因为会消除上面的全部数据，最后我采用的是：
    1.使用老毛桃安装恢复还原B硬盘上的操作系统，就是把iso文件放到老毛桃的ISO文件夹中，进入PE系统，谈出安装系统选项，然后将系统安装到原先的C盘，可能需要格式化或者是其他的东西。
    2.然后使用老毛桃重新修复B硬盘上的Windows引导。
    3.修复引导之后，重新在A硬盘上，使用U盘安装deepin系统，这样这个开机启动项就出现了Windows选项了，这样这个引导就重启起了作用。

    需要注意的一点是，如果启动windows之后，无法安装系统，需要重新进行123步，我就是尝试了三到四次重装系统，才最终把windows10安装成功的。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
