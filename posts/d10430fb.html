    Spring Boot解决中文乱码问题

    标签： Springboot 分类： Java 创建时间：2019-04-22 03:02:14 更新时间：2021-07-22 09:22:57

    中文乱码简直很蛋疼。如果在配置文件中application.properties编写了自定义配置，并且里面含有中文的话，因为他默认的会以ISO-8859-1编码格式读取，在控制台输出会出现乱码;或者是使用File读取文件的时候，会出现文件明明文件存在，但是总是报错显示文件不存在；调试的时候，在Idea代码显示的地方也是会显示乱码。

    这个时候网上会有很多解决方式，修改application.properties配置文件，添加编码设置等等，这里我设置的其实没有效果的。之说两者有效果的设置方式。

    1.读取配置文件时通过代码转换编码格式。

    就是默认时ISO编码，那就用ISO编码读取，然后在手动转换一下编码格式，当然，使用这种方式，既然已经读到了内容，转不转码，也随便了。

    2.修改Idea的设置。

    依次点击File -> Settings -> Editor -> File Encodings，将Properties Files (*.properties)下的Default encoding for properties files设置为UTF-8，将Transparent native-to-ascii conversion前的勾选上。

    重点强调的是，设置完成之后，一定要修改一下application.properties文件，然后保存一下，上面的设置才会生效，那怕你在空白处添加一个空格，然后删除也许。切记切记切记
