    1.前言

    最近又人让我帮忙找找有没有那种供销系统，管理进货出货的东西，于是我抽空对这几个类型的系统进行了整理。

    • CRM是管外部客户的——比如客户成单了没？销售今天去拜访客户了没？
    • OA则是管内部人员——比如考勤数据怎么样？申请请假怎么操作？
    • 而ERP是管内部物资——比如这个月工厂货物有多少出库、多少入库？

    3.赤龙ERP

    这个是我直接搜索ERP得到的，我查看了一下，有社区版和生态版。

    • 社区版
      社区版可以获取免费的授权，仅限个人学习或企业内部使用，包括了：采购/销售订单管理、入库/出库管理、库存管理、应收应付管理、付款/收款管理、子分类账与自动分录、凭证管理、主数据管理、用户与权限管理、职员与部门管理、数据字典、单点登录。

    • 生态版
      生态版是根据根据所选模块报价，比社区版本多了很多的功能：生产制造模块、费用报销模块 、采购/销售协议管理、协议订单与计划订单、库存盘点、库存调拨、质检管理、杂项出入库管理、出入库退货流程、预付款/预收款管理、借项/贷项通知单、收付款冲销、财务核算-日记账、过账、结账…等等，多了十几个模块，具体的功能，我对于整个的销售管理不是特别的清楚，具体到哪个功能，哪个地方这个模块又什么用，我其实一点不清楚的。

    我获取了免费的账号，进去之后看了看，很多我都不明白，也没有简单的增删改查之类的，我总是找不到入口，是不是体验版没有这些功能呢？就拿这个物料管理，里面连新增的按钮都没有，只能查询，其他的很多的东西也都是这个样子的。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
