    spring Boot升级过程中的坑

    标签： Springboot 分类： Java 创建时间：2019-01-09 06:28:05 更新时间：2025-04-28 14:37:41

    我在Deepin电脑上安装了openjdk11，用idea打开以前写过的一个Spring boot应用，这个应用是在windows下的orcale jdk8下写的。没有用到什么新的语法，开启debug，使用开启不了。出现了下面的问题：

    1.sun.misc包不存在。

    这个包可以说是orcale jdk自有的包，我应为在使用base64编码的时候，以及使用Signal信号的时候使用过，所以会出现报错。
    解决方法是：去掉了使用Signal的代码，将base64替换为java.utils中的base64方法。

    2.Disconnected from the target VM

    启动debug时，总是出现Disconnected from the target VM, address: ‘127.0.0.1:37119’, transport: ‘socket’ Process finished with exit code 1。应用起不来，还没有更多的错误信息。我把我自己在resource文件夹下的logback.xml配置文件删了，这个文件好像总是会阻止spring boot自带的错误显示出来，还是我配置的不对，不知道了。然后出现了下面的日志：

    重点在：java.lang.IllegalArgumentException: error at ::0 can’t find referenced pointcut pointCutMethod，我上网谷歌了下，aop-java-lang-illegalargumentexception-error-at-0-cant-find-referenced-poi 说的是，Aop版本不同，语法也不同。@Pointcut(“execution(* get*())”)换成@Pointcut(“execution(* get())”)这种方法不管事。有人说是JDK不匹配，如果要使用AspectJ完成注解切面需要注意下面的JDK与AspectJ的匹配：

    　　JDK1.6 —— aspectJ1.6
    　　JDK1.7 —— aspectJ1.7.3+

    修改了一下pom.xml确实管事，将spring-boot-starter-aop这个依赖注释掉，因为我安装的是openjdk11,所以我用了最新的aspectjweaver

    终于算是解决了兼容性问题。这个问题不知到归类到哪篇文章好，就干脆写到这里吧，

    4.Spring-mail的时候配置qq邮箱

    使用Spring-mail的时候配置qq邮箱时，应该填写port：465端口，否则会报错。

    5.ApplicationEventMulticaster not initialized

    spring boot 启动报错如下：
    ApplicationEventMulticaster not initialized - call ‘refresh’ before multicasting events via the context: org.springframework.boot.context.embedded.AnnotationConfigEmbeddedWebApplicationContext@7e3181aa: startup date [Fri Jul 21 10:40:28 CST 2017]; parent:
    这个问题我觉得很头疼，刚刚还能启动的系统，啥都没改，就报了上面的错，启动不了。我把spring-boot-starter-aop依赖换上面的，系统启动起来了。然后我在把aop换回来，这次系统启动又不报错误了。真是蛋疼啊。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
