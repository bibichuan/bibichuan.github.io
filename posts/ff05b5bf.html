    这是我的第一篇个人博客。主要是练习用hexo和github编辑自己的文章。主要编写些什么东西呢？随便啦。

    第一个方面就是写些自己在技术上遇到的问题以及解决方法，第二个就是编写一些生活的感悟吧，瞎逼逼一下。

    就像许多普通的开发者一样，我从小白开始，也一直在小白的层面挣扎。对于非科班（本科不是计算机专业）出身的程序员，从最开始学校里面的c语言到c#,工作了之后便开始了前端修行，期间玩过linux,写过java,练习过python,可谓门门了解，却又门门不精。牛逼的人，在刚毕业的时候，说的再往前一点，从小就开就开始牛逼了，像我这种没啥优点的，浑水摸鱼了很多年，却还是一个一窍不通的萌新（鄙视自己），不知道还能在程序员这一行挣扎多久，只能说加油吧，骚年。我不得不承认，自己真的就是间歇性踌躇满志，持续性混吃等死。

    这个社会，其实是个精英的社会，特别是北上广深一二线大城市，如果你不能在一个行业成为一个领导者，注定是生活在社会的底层的。

    我爱抱怨生活，我也时长的告诫自己不能抱怨生活，任何事情都是自己的选择。少壮不努力，老大徒伤悲，说的就是我吧。人生，不过是一个因果罢了，前日的因，造就了今日的果。我想最好的处理方法就应该是随缘吧。我有很多的梦想，想要有钱，想要做自己喜欢的工作，想要能帮家里点忙，让父母提前退休，可是如今已经到了而立之年，依然是买不起房，买不起车，在国家的贫困线以下徘徊，在杭南漂。其实我最大的心愿，就是有一个属于自己的房子，在自己喜欢的城市里，安安静静的为国家、为社会、为家庭，付出自己的一切，曾经豪情万丈，曾经想仗剑走天涯，后来，因为bug多，没去。

    兴趣是最大的动力，坚持是最好的途径。找到自己今生愿意为之付出一切的事业，并不断的克服一切，努力向上，这是一种幸福。哪怕最后什么都没得到，空无一物，起码我们有值得让自己刻骨铭心的记忆。无论你想成功也罢，不想成功也罢，总逃不过一个难字。事事难，实时难，处处难，想要过的舒服，就要不断的不断的不断的克服一个又一个的困难。

    isNaN 和 Number.isNaN 函数的区别？

    1.函数 isNaN 接收参数后，会尝试将这个参数转换为数值，任何不能被转换为数值的的值都会返回 true，因此非数字值传入也会返回 true ，会影响 NaN 的判断。

    2.函数 Number.isNaN 会首先判断传入参数是否为数字，如果是数字再继续判断是否为 NaN ，不会进行数据类型的转换，这种方法对于 NaN 的判断更为准确。

    为什么0.1+0.2 ! == 0.3，如何让其相等?

    一个直接的解决方法就是设置一个误差范围，通常称为“机器精度”。对JavaScript来说，这个值通常为2-52，在ES6中，提供了Number.EPSILON属性，而它的值就是2-52，只要判断0.1+0.2-0.3是否小于Number.EPSILON，如果小于，就可以判断为0.1+0.2 ===0.3。

    == 操作符的强制类型转换规则？

    1.首先会判断两者类型是否**相同，**相同的话就比较两者的大小。

    2.类型不相同的话，就会进行类型转换。

    3.会先判断是否在对比 null 和 undefined，是的话就会返回 true。

    4.判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number。

    5.判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断。

    6.判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断。
